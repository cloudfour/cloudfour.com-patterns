# Button

The `c-button` class may be applied to button and link elements. The button appears filled by default so that the actionable element is clearly distinguished from surrounding text.

## Styles

Some buttons may not require as much emphasis as others. The `c-button--secondary` and `c-button--tertiary` modifiers reflect lesser importance with reduced contrast.

## Button with icon

Use an `iconStart` or `iconEnd` block to add an [Icon component](/docs/components-icon--basic) before or after the Button `content` block.

```twig
{% embed '@cloudfour/components/button/button.twig' %}
  {% block iconStart %}
    {% include '@cloudfour/components/icon/icon.twig' with { name: 'notifications', inline: true } %}
  {% endblock %}
  {% block content %}Get notifications{% endblock %}
{% endembed %}

{% embed '@cloudfour/components/button/button.twig' %}
  {% block content %}Button with icon{% endblock %}
  {% block iconEnd %}
    {% include '@cloudfour/components/icon/icon.twig' with { name: 'plus', inline: true } %}
  {% endblock %}
{% endembed %}
```

## Template Properties

- `class`: Append a class to the root element.
- `iconStart` (block): Can be used to add an [Icon component](/docs/components-icon--basic) before the Button `content` block.
- `content` (block): Block to add Button elements.
- `iconEnd` (block): Can be used to add an [Icon component](/docs/components-icon--basic) after the Button `content` block.
