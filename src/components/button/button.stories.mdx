# Button

The `c-button` class may be applied to button and link elements. The button appears filled by default so that the actionable element is clearly distinguished from surrounding text.

## Styles

Some buttons may not require as much emphasis as others. The `c-button--secondary` and `c-button--tertiary` modifiers reflect lesser importance with reduced contrast.

## Button with icon

The recommended way to add an icon to a button is to use the `icon_start` or
`icon_end` Button component argument. You can pass the name of any of
the [available icons](/docs/design-icons--page).

```twig
{% embed '@cloudfour/components/button/button.twig' with { icon_start: 'notifications' } %}
  {% block content %}Get notifications{% endblock %}
{% endembed %}

{% embed '@cloudfour/components/button/button.twig' with { icon_end: 'search' } %}
  {% block content %}Find it!{% endblock %}
{% endembed %}
```

### Advanced usage

If you need a custom icon, you can pass an `<svg>` element directly via the
`icon_start` or `icon_end` Button component block.

This option is only recommended in very unique cases. Also note the usage of the
`c-icon c-icon--inline` CSS classes on the `<svg>` element.

```twig
{% embed '@cloudfour/components/button/button.twig' %}
  {% block content %}Button with a custom icon{% endblock %}
  {% block icon_end %}
    <svg viewBox="0 0 24 24" width="24" height="24" class="c-icon c-icon--inline">
      <circle r="8" cx="12" cy="12"></circle>
    </svg>
  {% endblock %}
{% endembed %}
```

## Template Properties

- `class`: Append a class to the root element.
- `icon_start`: Pass the name of any [available icon](/docs/design-icons--page) to add the icon _before_ the Button `content` block.
- `icon_end`: Pass the name of any [available icon](/docs/design-icons--page) to add the icon _after_ the Button `content` block.

## Template Blocks

- `icon_start`: Can be used to render a custom `<svg>` element before the Button `content` block.
- `content`: Block to add Button elements.
- `icon_end`: Can be used to render a custom `<svg>` element after the Button `content` block.
